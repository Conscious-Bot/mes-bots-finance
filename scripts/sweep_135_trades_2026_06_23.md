## Lessons apprises

1. **TG `/position_sell` parser** : `qty price reasoning` — currency NEVER taken
   comme arg. Pour JPY/USD/KRW, le price doit être en NATIVE units (¥/¢/$/₩),
   pas EUR. Si tu types `172 EUR`, le EUR est juste un mot, et 172 est traité
   comme native ¥/€/$ selon ticker.

2. **Append-only audit trail** : transactions table a triggers `writeonce_update`
   + `writeonce_delete` qui RAISE ABORT. Toute correction doit passer par
   compensating tx (BUY pour reverse SELL, SELL pour reverse BUY) ou ADJUST tx
   (override price + fx via JSON notes `target_tx_id`).

3. **CHECK qty > 0** sur transactions : les ADJUST tx doivent avoir qty
   placeholder=1 (la valeur est ignorée par `compute_pmp_realized` qui ne lit
   que `price_native` + `fx_at_trade` des ADJUST). Pas qty=0 (CHECK fail).

4. **L28 entry_value immutable** : trigger DB enforce write-once sur thèses
   entry_value/currency/asof. Toute correction de cost basis doit passer par
   ADJUST tx au niveau ledger, pas UPDATE thèse. C'est protection structurelle
   contre revision history-mistake.

5. **Single source = VM** (acted 23/06) : trades via Telegram VM ou SSH+python
   sur VM. JAMAIS sur Mac local. Sync VM→Mac hourly launchd. Si user log
   Telegram + claude log SSH = double-fire phantom = need cure.
