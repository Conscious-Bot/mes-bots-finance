## Discipline lessons codified (Lessons 28-34)

7 lessons in 1 session — record for this project. Each maps to an empirical violation during the session:

| Lesson | Codifies |
|---|---|
| **L28** | Pre-commit gates order: ruff check `.` BEFORE pytest -q |
| **L29** | Code-patching scripts must `assert`/`sys.exit(1)` hard-fail, not silent skip |
| **L30** | Test log capture: monkeypatch module's `_log` reference, not pytest caplog |
| **L31** | Schema audit (`PRAGMA table_info`) BEFORE writing invariant assertions |
| **L32** | Compound Python script edits must be ATOMIC (validate all anchors, then write) |
| **L33** | Bot running = SQLite write lock contention — kill+verify+write+restart+verify pattern |
| **L34** | Gates must HARD-FAIL before commit (shell exit code on ruff, never assume "last gate OK") |

Combined effect: discipline gap → codified rule → next refactor uses gate hard-fail pattern → no recurrence.
